使用路由

1、url匹配模式

# urls.py
from django.urls import path
from . import views
urlpatterns = [
    path('articles/2003/', views.special_case_2003),
    path('articles/<int:year>/', views.year_archive),
    path('articles/<int:year>/<int:month>/', views.month_archive),
    path('articles/<int:year>/<int:month>/<slug:slug>/', views.article_detail),
]

对 /articles/2005/03/ 的请求将与列表中的第三个条目匹配。   
Django会调用函数views.month_archive（request，year=2005，month=3）

2、指定函数默认值

# urls.py
urlpatterns = [
    path('blog/', views.page),
    path('blog/page<int:num>/', views.page),
]

# View (in blog/views.py)
def page(request, num=1):
    # Output the appropriate page of blog entries, according to num.

3、包含其它的URLconfs

3.1项目中主urls.py包含应用的urls.py

path('community/', include('aggregator.urls')),

3.2去除冗余,包含当前应用中的路由

path('index/<int:page>',views.hello),
path('index/<str:page>',views.page1,{'page':'helloworld'}),
path('index/<str:page>/<int:page1>',views.second)

可以简写成：

path('index/',include([
    path('<int:page>',views.hello),
    path('<str:page>',views.page1,{'page':'helloworld'}),
    path('<str:page>/<int:page1>',views.second)
])),

3.3参数的传递

被包含的URLconf 会收到来自父URLconf 捕获的任何参数

4、传递额外的参数给视图函数

path('blog/<int:year>/', views.year_archive, {'foo': 'bar'}),

对于/blog/2005/的请求，Django将调用views.year_archive（request，year=2005，foo='bar'）

传递额外的参数到include
    path('blog/', include('inner'), {'blog_id': 3}),

# inner.py
urlpatterns = [
    path('archive/', views.archive),
    path('about/', views.about),
]

额外参数blog_id 会传到inner中的每一个urlconfs，即会传入`archive`函数和`about`函数

5、url反向解析

  • 在模板中：使用url模板标记。
  • 在Python代码中：使用reverse()函数。
为路由定义一个name
#urls.py
path('articles/<int:year>/', views.year_archive, name='news-year-archive'),
模板中解析
<a href="{% url 'news-year-archive' 2012 %}">2012 Archive</a>
python代码中使用反向解析
def year_archive(request,year):
    return HttpResponseRedirect(reverse('news-year-archive', args=(year,)))

6、命名空间

命名空间的反向解析：
reverse("<namespace>:<url-name>", kwargs={"<kwarg>": "<val>"})
URL中使用
{% url "namespace:url-name" kwarg=val %}
命名空间的定义：
#posts/urls.py
app_name = 'posts'                 #通过app_name 来定义名称空间
urlpatterns=[
    path('articles/<int:year>/', views.year_archive, name='news-year-archive'),
]
模板中使用
<a href="{% url 'posts:news-year-archive' 2012 %}">2012 Archive</a>
python代码中：
def year_archive(request,year):
    return HttpResponseRedirect(reverse('posts:news-year-archive', args=(year,)))
